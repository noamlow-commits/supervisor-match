# מאגר מדריכי הדרכה — בית הספר הדיאלוגי

אפליקציית web לשידוך בין מדריכים לתלמידים. דירקטוריון של ~10 מדריכים, ~10-30 תלמידים, פעם-פעמיים בשנה.

**🔗 אתר חי:** https://noamlow-commits.github.io/supervisor-match/

## חמשת המסכים

| מסך | למי | URL |
|---|---|---|
| `index.html` | תלמידים — דירקטוריון + סינון | `/` |
| `supervisor.html` | מדריך — עריכת פרופיל אישי | `/supervisor.html?token=...` |
| `register.html` | מדריך חדש — הרשמה עצמית | `/register.html` |
| `claim.html` | מדריך עם פרופיל מוכן (לרוב מבוגרים) — כניסה ישירה ע״י שם+מייל | `/claim.html` |
| `admin.html` | מזכירה אקדמית + צוות הניהול — דשבורד | `/admin.html` |

## שלוש דרכי כניסה למדריכים

### דרך A: הרשמה עצמית (`register.html`)
מדריך חדש מקבל מהצוות **קוד הזמנה** (ברירת מחדל `CFGUSH26`), נרשם לבד, מקבל מייל עם קישור אישי.

### דרך B: כניסה לפרופיל מוכן (`claim.html`) — חדש!
**מותאם למדריכים מבוגרים שמתקשים למלא טופס.**
1. הצוות ממלא טבלה ב-Sheet (טאב `BulkImport`) עם נתוני המדריך
2. הצוות מריץ `bulkImportFromSheet()` ב-Apps Script
3. המערכת יוצרת פרופיל מוכן ושולחת מייל למדריך
4. המדריך נכנס ל-`claim.html`, מזין שם+מייל, ונכנס ישר לעריכה

### דרך C: עריכה דרך הקישור האישי
המדריך פותח את הקישור האישי שקיבל במייל ועורך ישירות.

## הקמה — צעד-צעד (התקנה ראשונית)

### 1. צור Google Sheet חדש
פתח https://sheets.google.com → גיליון חדש בשם "מאגר מדריכים".

### 2. הדבק את קוד ה-Apps Script
- Extensions → Apps Script → מחק הקוד הקיים → הדבק את `apps-script-code.js` → שמור

### 3. הרץ setupSheets פעם אחת
- בתפריט הפונקציות בחר `setupSheets` → Run → אשר הרשאות
- ב-Sheet נוצרים 4 טאבים: Supervisors, Parameters, Settings, BulkImport

### 4. פרסם כ-Web App
- Deploy → New deployment → ⚙ → Web app
- Execute as: **Me**, Who has access: **Anyone**
- Deploy → העתק URL (גומר ב-`/exec`)

### 5. הגדר את appBaseUrl ב-Sheet
בטאב Settings → שורת appBaseUrl → שים `https://noamlow-commits.github.io/supervisor-match/`

### 6. חבר את האפליקציה ל-Backend
פתח כל אחד מהדפים → ⚙ בפינה השמאלית → הדבק URL.

## ייבוא מדריכים בכמות (Bulk Import)

### קיצור: ייבוא 16 מדריכי הדיאלוגי בלחיצה
אם את/ה מתחיל/ה את העונה ורוצה לטעון את 16 מדריכי בית הספר הדיאלוגי
(נשאבו אוטומטית מ-jewishpsychology.org/hadialogy ו-bogrim_h.php):

1. ב-Apps Script → תפריט פונקציות → `seedHadialogyDirectory` → Run
2. הפונקציה ממלאת 16 שורות בטאב BulkImport (מדלגת על מיילים שכבר קיימים)
3. עברו על השורות בטאב BulkImport, השלימו/תקנו מה שחסר (הסבר על הסגנון, התמחויות וכו')
4. הריצו `bulkImportFromSheet` — נשלחים מיילים ל-16 המדריכים

### זרימה מומלצת: למלא קודם → לשלוח אחר כך
הצוות עורך את הפרופילים על דעת עצמו, ורק אחר כך שולחים מייל למדריכים
שיעברו ויתקנו (חוסך זמן למדריכים מבוגרים).

1. הריצו `seedHadialogyDirectory` — ממלא 16 שורות ב-BulkImport
2. הריצו `bulkImportSilent` — יוצר רשומות מדריך ב-Supervisors **בלי לשלוח מייל**
   (כל רשומה נוצרת עם `mailed=FALSE`)
3. עורכים פרופילים בעצמכם — שתי דרכים נוחות:
   - **claim.html** עם השם והמייל של המדריך → נכנסים לעריכה
   - או: ב-Sheet → טאב Supervisors → מעתיקים `token` → `supervisor.html?token=XXX`
4. כשמוכנים — הריצו `sendInvitationsToUnmailed` → שולח מייל לכל מי ש-`mailed=FALSE`
   ומסמן `mailed=TRUE`. בטוח להריץ פעמיים: לא ישלח שוב.

⚠ **בטיחות:** רשומות ישנות (לפני שדה `mailed`) מקבלות `mailed=TRUE` כברירת מחדל
כדי שלא יישלחו להן מיילים בטעות. הפונקציה `sendInvitationsToUnmailed` שולחת
**רק** למי ש-mailed הוא מפורשות FALSE.

### השלבים הידניים
1. ב-Sheet → טאב **BulkImport**
2. מלא/י שורה לכל מדריך:
   | עמודה | חובה | דוגמה |
   |---|---|---|
   | imported | אוטומטי | (השאר ריק/FALSE) |
   | fullName | ✓ | ד״ר ישראל ישראלי |
   | email | ✓ | israel@example.com |
   | credential | מומלץ | פסיכולוג קליני, MA |
   | yearsSupervising | מומלץ | 15 |
   | orientations | מומלץ | דינאמי;CBT |
   | populations | מומלץ | מבוגרים;נוער |
   | specialties | מומלץ | טראומה;חרדה ודיכאון |
   | styleText | מומלץ | תיאור קצר של הסגנון |
   | format | מומלץ | היברידי |
   | area | מומלץ | ירושלים |
   | phone | מומלץ | 0501234567 |
   | whatsappEnabled | אופציונלי | TRUE/FALSE |
   | hasSpot | אופציונלי | TRUE (ברירת מחדל) |
   | autoPublish | אופציונלי | TRUE = פרסם מיד / FALSE = השאר כטיוטה |
3. ב-Apps Script → תפריט פונקציות → `bulkImportFromSheet` → Run
4. כל מדריך מקבל מייל אוטומטית עם קישור ל-claim.html
5. עמודת `imported` תסומן TRUE — שורות שכבר עובדו לא יוקפצו פעמיים

### מה המדריך עושה
1. מקבל מייל "הפרופיל שלך מוכן"
2. נכנס ל-claim.html
3. מזין שם מלא + מייל (כפי שמסר/ה)
4. נפתח לו הפרופיל המוכן
5. בודק/ת, מתקן/ת אם צריך, לוחץ/ת "פרסם"

## שדרוג מ-v1 (אם הקמת קודם)

1. החלף את `apps-script-code.js` ב-Apps Script editor
2. הרץ ידנית את `migrateToV2()` (במקום `setupSheets`)
3. **פרוס מחדש:** Deploy → Manage deployments → ✏ → New version → Deploy

## דשבורד הנהלה (admin.html)

מסך **רק** למזכירה האקדמית. מוגן ב-`adminPin`.

- כרטיסי סיכום (סך מדריכים, תלמידים שובצו, ממתינים, מקומות פנויים)
- סרגל התקדמות שיבוץ באחוזים
- טבלת מדריכים עם מספר תלמידים אצל כל אחד

**מי רואה מה:**
- תלמיד: רק "יש מקום / אין מקום"
- מדריך: רק את המספר של עצמו
- מזכירה: הכל

## עריכת פרמטרים בלי קוד
טאב Parameters ב-Sheet — `שדה | אפשרות`. שינויים נכנסים בריענון.
פרמטרים: orientations, populations, specialties, formats, areas

## שדות בפרופיל מדריך

### חובה (11 שדות)
1. שם מלא  2. תואר/הסמכה  3. שנות הדרכה
4. **סוגי טיפול:** דינאמי / CBT / טיפול ממוקד טראומה - SE/EMDR / טיפול זוגי / מיניות / התמכרויות
5. אוכלוסיות  6. תחומי התמחות (עד 5)
7. על הסגנון (50-400 תווים)
8. פורמט  9. אזור  10. זמינות שבועית
11. סטטוס מקומות

### דרכי קשר
טלפון נייד + checkbox WhatsApp / מייל

### שדה ניהול (פרטי, רק למזכירה)
כמה תלמידים אישרת לקבל השנה

## עקרונות עיצוב
- דירקטוריון, לא מנוע שידוך
