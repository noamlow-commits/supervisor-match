# מאגר מדריכי הדרכה — בית הספר הדיאלוגי

אפליקציית web לשידוך בין מדריכים לתלמידים. דירקטוריון של ~16 מדריכים, ~10-30 תלמידים, פעם-פעמיים בשנה.

**🔗 אתר חי:** https://noamlow-commits.github.io/supervisor-match/

## חמשת המסכים

| מסך | למי | URL |
|---|---|---|
| `index.html` | תלמידים — דירקטוריון + סינון | `/` |
| `supervisor.html` | מדריך — עריכת פרופיל אישי | `/supervisor.html?token=...` |
| `register.html` | מדריך חדש — הרשמה עצמית | `/register.html` |
| `claim.html` | מדריך עם פרופיל מוכן (לרוב מבוגרים) — כניסה ע״י שם+מייל | `/claim.html` |
| `admin.html` | מזכירה אקדמית + צוות הניהול — דשבורד **+ עריכה לכל פרופיל** | `/admin.html` |

## שלוש דרכי כניסה למדריכים

### דרך A — הרשמה עצמית (`register.html`)
מדריך חדש מקבל קוד הזמנה (`CFGUSH26`), נרשם לבד, מקבל מייל עם קישור אישי.

### דרך B — כניסה לפרופיל מוכן (`claim.html`)
**מותאם למדריכים מבוגרים שמתקשים למלא טופס.** הצוות יוצר פרופיל מראש, המדריך נכנס עם שם+מייל בלבד.

### דרך C — קישור אישי במייל
המדריך פותח את הקישור האישי שקיבל ועורך ישירות.

## הקמה — צעד-צעד

### 1. צור Google Sheet חדש
פתח https://sheets.google.com → גיליון חדש בשם "מאגר מדריכים".

### 2. הדבק את קוד ה-Apps Script
Extensions → Apps Script → מחק הקוד הקיים → הדבק `apps-script-code.js` → שמור.

### 3. הרץ setupSheets פעם אחת
תפריט הפונקציות → `setupSheets` → Run → אשר הרשאות.
נוצרים 4 טאבים: `Supervisors`, `Parameters`, `Settings`, `BulkImport`.

### 4. פרסם כ-Web App
Deploy → New deployment → ⚙ → Web app → Execute as: **Me**, Who has access: **Anyone** → Deploy → העתק URL (גומר ב-`/exec`).

### 5. הגדר את appBaseUrl ב-Sheet
Settings → `appBaseUrl` → `https://noamlow-commits.github.io/supervisor-match/`

### 6. חבר את האפליקציה ל-Backend
פתח כל אחד מהדפים → ⚙ בפינה השמאלית → הדבק URL.

## ייבוא בכמות (Bulk Import) — שני שלבים

### Phase 1: יצירה שקטה
1. ב-Sheet → טאב **BulkImport** — מלא/י שורה לכל מדריך (או הרץ/י `seedHadialogyDirectory`)
2. הרץ/י `bulkImportSilent()` ב-Apps Script
3. נוצרים פרופילים ב-Supervisors עם `mailed=FALSE`. **לא נשלחים מיילים עדיין**

### Phase 2: השלמה ידנית (אופציונלית)
לפני שמודיעים למדריכים, אפשר למלא ידנית את הפרופיל של המדריכים המבוגרים בטאב Supervisors. **בקלות יותר עכשיו (v2.4)**: כנסי ל-`admin.html`, לחצי "✎ ערוך" ליד מדריך, ערכי מהמסך.

### Phase 3: שליחת הזמנות
הרץ/י `sendInvitationsToUnmailed()` — שולח מייל רק לאלה ש-`mailed=FALSE`, ומסמן TRUE.

## שדרוג מ-v1 / v2

`migrateToV2()` עושה:
- מעדכן רשימת `orientations` ל-6 הסוגים החדשים
- ממפה אוריינטציות ישנות (פסיכודינמי → דינאמי)
- מוסיף `adminPin`, `expectedStudents` ל-Settings
- מוסיף עמודות `studentsAccepted`, `mailed` ל-Supervisors
- יוצר טאב `BulkImport`

**⚠ אחרי כל שינוי קוד** — חובה לפרוס מחדש: Deploy → Manage deployments → ✏ → New version → Deploy.

## דשבורד הנהלה (admin.html)

מסך **רק** למזכירה אקדמית. מוגן ב-`adminPin`.
- **כרטיסי סיכום:** סך מדריכים, תלמידים שובצו, ממתינים, מקומות פנויים
- **סרגל התקדמות שיבוץ באחוזים** — מבוסס על רשימת תלמידים אמיתית (v2.5)
- **ניהול תלמידים (v2.5)** — הוספת תלמיד יחיד, הדבקת רשימה רב-שורתית, שיוך כל תלמיד למדריך מתוך dropdown, מחיקת תלמיד. בכל רגע רואים מי שובץ ומי ממתין — ברמת השם.
- **טבלת מדריכים** עם מספר תלמידים אצל כל אחד (נספר אוטומטית מרשימת התלמידים אם קיימת; אחרת מהדיווח העצמי של המדריך)
- **כפתור "✎ ערוך" לכל מדריך (v2.4)** — פותח את טופס העריכה של אותו מדריך בלשונית חדשה. המזכירה יכולה לעדכן/לתקן פרופילים בלי לבקש מהמדריך לעשות זאת בעצמו.

### זרימת עבודה (v2.5 + v2.6)
1. בתחילת המחזור: המזכירה מדביקה את רשימת התלמידים (שם בכל שורה, או "שם, מייל, טלפון") בלוח האדמין
2. עם כל שיבוץ — שתי דרכים מקבילות:
   - **המדריך** נכנס לפרופיל שלו (`supervisor.html?token=...`), בסקציית "התלמידים שלי" בוחר תלמיד מהרשימה הנפתחת ולוחץ "אני לוקח/ת"
   - **המזכירה** משייכת בעצמה דרך ה-dropdown בטבלת התלמידים בדשבורד
3. בכל רגע נתון: כרטיס "תלמידים ממתינים" מציג מספר + מי הם בשמות. הטבלה ממוינת כך שהממתינים בראש

**מי רואה מה:**
- תלמיד: רק "יש מקום / אין מקום" (לא רואה את הרשימה)
- מדריך: רק את המספר של עצמו
- מזכירה: הכל + רשימת תלמידים מלאה + עריכה מלאה לכל פרופיל

## Stack
HTML/CSS/JS סטטי + Google Apps Script + Google Sheets + GitHub Pages.
פונט: Assistant. אין build, אין npm. API: GET + fetch() עם 12s timeout.

## עקרונות עיצוב
- דירקטוריון, לא מנוע שידוך
- מינימום התערבות צוות *(אבל המזכירה כן יכולה להתערב כשצריך)*
- אין פופולריות (תלמידים לא רואים מונים, מדריכים לא רואים אחד את השני)
- מובייל-first
- שלוש דרכי כניסה למדריכים
- ייבוא דו-שלבי (יצירה → השלמה → הזמנה)

## דמו vs חי
- דמו (אין URL): localStorage + מדריכי דמו, תווית "דמו"
- חי (URL מוגדר): Google Sheet, מיילים אמיתיים, ללא תווית

## גרסאות
- **v1** — דירקטוריון בסיסי + הרשמה עצמית
- **v2** — דשבורד הנהלה, סוגי טיפול מפושטים, studentsAccepted, מיגרציה
- **v2.1** — claim.html + bulk import
- **v2.2** — seedHadialogyDirectory + ייבוא דו-שלבי שקט
- **v2.3** — הסרת "תחומי התמחות" מהממשק, תיקון באג הכפלת צ׳יפים
- **v2.4** — מזכירה יכולה לערוך כל פרופיל מהדשבורד (כפתור "ערוך" ליד כל מדריך)
- **v2.5** — ניהול רשימת תלמידים בלוח האדמין: שיוך תלמיד למדריך, מעקב מי שובץ ומי ממתין ברמת השם, הדבקת רשימה רב-שורתית. סך התלמידים מחושב אוטומטית מהרשימה (הגדרת `expectedStudents` הופכת לפולבק כשהרשימה ריקה)
- **v2.6** — שמירה אוטומטית בפרופיל המדריך + פנל "חסר עוד למלא". שני שדרוגים בקובץ אחד:
  - **שמירה אוטומטית (debounced 800ms)** — כל שינוי בפרופיל נשמר לשרת ללא צורך בלחיצת כפתור. סטטוס בראש העמוד: "שומר…" / "✓ נשמר אוטומטית" / "✗ שמירה נכשלה". מטרה: לאפשר למזכירה האקדמית לעזור למדריכים פחות-טכנולוגיים למלא את הפרופיל בשלבים בלי חשש מאיבוד נתונים
  - **פנל "חסרים שדות"** — מעל כפתורי השמירה, פנל חי המראה בכל רגע אילו שדות עדיין חסרים לפרסום (כולל מונה תווים לתיאור הסגנון). הופך לירוק עם "✓ הפרופיל מוכן לפרסום" כשהכל מולא
  - אין שינויים שוברי תאימות — קוד קיים ופונקציות שמירה/פרסום עובדים כרגיל
- **v2.7** — פרסום פרופיל חלקי + הפרדה ויזואלית בלוח התלמידים (frontend בלבד):
  - **פרסום פרופיל חלקי** — כפתור "פרסם פרופיל" כבר לא חוסם כשחסרים שדות. במקום זה, מציג confirm עם רשימת השדות החסרים ושאלה "לפרסם בכל זאת?". מטרה: לאפשר למזכירה למלא פרופיל "תוך כדי תנועה" — לפרסם את מה שיש כדי שתלמידים יראו, ולהשלים שדות אחר כך
  - **הפרדה בין ממתינים ושובצו** — בטבלת התלמידים בלוח האדמין, שתי קבוצות-משנה ויזואליות: "⏳ ממתינים לשיבוץ" (רקע צהוב, עם מונה) ו-"✓ שובצו" (רקע ירוק בהיר, עם מונה). שורות ממתינים בולטות יותר; סטטוס בכל שורה הופך ל-pill צבעוני (צהוב/ירוק) במקום טקסט פשוט. סיכום בראש הטבלה צבוע בהתאם
- **v2.6** — מדריך יכול לסמן בעצמו "אני לוקח/ת תלמיד X" מתוך פרופיל העריכה שלו (`supervisor.html?token=...`). סקציה חדשה "התלמידים שלי" מציגה רשימה של תלמידים שעדיין מחפשים ומאפשרת לבחור מתוכם. השיבוץ נכנס מיידית לדשבורד המזכירה. אימות ע"י הטוקן הקיים, ללא סיסמה נוספת. הגנה על race condition: אם שני מדריכים מנסים לסמן אותו תלמיד, השני מקבל שגיאה
