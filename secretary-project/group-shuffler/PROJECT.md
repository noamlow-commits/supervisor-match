## מטרה
כלי שמשבץ רשימת משתתפים ל**זוגות לימוד** רנדומליים, מצמצם חזרות לאורך זמן, ומנהל **סבבים של חודשיים** (כל זוג לומד חודשיים ואז מתחלף). קיימת **קבוצה קבועה** ("הגברים") שאינה מתחלפת ומוצגת בכל סבב כקבוצה אחת.

## החלטות יסוד (נועם)
| נושא | החלטה |
|---|---|
| יחידת הלימוד | **זוגות** (2). מספר אי-זוגי → שלישייה אחת, שמתחלפת בהוגנות |
| קבוצה קבועה | **"הגברים"** — רשימה נפרדת, אינה מתערבבת, מוצגת כקבוצה אחת בכל סבב (2026-07-07) |
| תדירות החלפה | **חודשיים** לכל זוג (מלבד הקבוצה הקבועה). `ROTATION_MONTHS=2` ב-config |
| שיבוץ | **רנדומלי + מזעור חזרות** — שומר היסטוריה, מנסה לא לחזור על אותו זוג |
| ארכיטקטורה | Apps Script + Google Sheet (כמו שאר אפליקציות המכון) |
| מזהה DB | ב-**Script Properties** מההתחלה (`GS_DB_ID`) — עמיד בהדבקת config |

## איך זה עובד
1. **משתתפים לשיבוץ בזוגות** — שם בכל שורה → "שמירת רשימה". נשמר בלשונית `תלמידות`.
2. **הגברים — קבוצה קבועה** — רשימה נפרדת → "שמירת הקבוצה הקבועה". נשמר בלשונית `קבוצה קבועה`. מי שכאן מוחרג משיבוץ הזוגות.
3. **סבב חדש** — "שבץ זוגות" → רואים את הזוגות + הקבוצה הקבועה (בצבע נבדל) → "ערבב שוב", **החלפה ידנית**, או "שמור סבב".
   - **החלפה ידנית (2026-07-09):** לוחצים על שם ואז על שם אחר — והם מתחלפים זה בזה בין הזוגות. מתקנים זוג-שניים בלי לערבב הכול מחדש. עובד גם על טיוטה שטרם נשמרה וגם על **הסבב הנעול** (כפתור "✏️ החלפה ידנית בסבב הנוכחי" → "💾 שמור שינויים"); בעריכת סבב נעול התאריכים נשמרים ורק הזוגות מתעדכנים (API `apiUpdateRound`/`updateRound_`, עמודה 4 בלשונית `סבבים`).
4. **שמירת סבב** — נשמר בלשונית `סבבים` עם תאריך התחלה + **תאריך החלפה (התחלה + חודשיים)** + הזוגות + צילום הקבוצה הקבועה. מכאן מתחילה ספירת החודשיים.
5. **סטטוס הסבב** — המסך מציין כמה ימים נשארו עד ההחלפה, ומתריע 🔁 כשהגיע הזמן.
6. **היסטוריה** — כל הסבבים עם טווח התאריכים; אפשר למחוק.

### האלגוריתם (`generatePairs_` ב-lib.gs)
- `pairCounts_` — מכל הסבבים: כמה פעמים כל שני משתתפים היו זוג.
- מפריד מראש את **הקבוצה הקבועה** (לא נכנסת לזוגות).
- מספר אי-זוגי → מפריש **איש-שלישייה** = מי שהיה בשלישייה הכי מעט בעבר (`trioCounts_`, שובר-שוויון אקראי) — כך אותו אדם לא נתקע בשלישייה כל פעם.
- בונה זוגות טהורים: לכל אחד מצרף את הפנוי שאיתו היה **הכי פחות** (שובר-שוויון אקראי) → מגוון לאורך זמן.
- משבץ את איש-השלישייה לזוג שאיתו היה הכי פחות.

## פריסה (פרויקט Apps Script נפרד, תחת חשבון המכון)
1. פרויקט Apps Script **חדש**; הדבק את 4 קבצי ה-.gs + `index.html`.
2. הרץ **`setup()`** פעם אחת → נוצר הגיליון "שיבוץ ללימוד בזוגות — App DB" (לשוניות `תלמידות`/`קבוצה קבועה`/`סבבים`), id נשמר ב-Script Properties.
3. **Deploy → New deployment → Web app** (Execute as: Me · Access: לפי הצורך).
4. פתח את /exec → הזן שמות (+ קבוצה קבועה) → שבץ → שמור סבב.

## רעיונות להמשך (אופציונלי)

- מטריצת כיסוי: כמה פעמים כל אחד כבר נפגש עם כל אחד.
- "נעילת" זוגות שאסור/חובה יחד.
