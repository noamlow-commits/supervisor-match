# שיבוץ ללימוד בזוגות

תת-פרויקט של `secretary-project/`. אפליקציית web פשוטה (Apps Script + Google Sheet) שמשבצת רשימת משתתפים ל**זוגות לימוד** רנדומליים, מצמצמת חזרות, ומנהלת סבבים של **חודשיים**. יש **קבוצה קבועה** ("הגברים") שאינה מתחלפת.

## מה היא עושה
- **משתתפים לשיבוץ בזוגות** — מזינים שם בכל שורה, שומרים.
- **הגברים — קבוצה קבועה** — רשימה נפרדת שאינה מתערבבת; מוצגת כקבוצה אחת בכל סבב, ומוחרגת משיבוץ הזוגות.
- **שיבוץ** — "שבץ זוגות" → רואים את הזוגות + הקבוצה הקבועה → "ערבב שוב" או "שמור סבב".
- **שיבוץ חכם** — מעדיף להרכיב מי שעדיין לא היו יחד (לפי היסטוריית הסבבים) → מזעור חזרות. מספר אי-זוגי → שלישייה אחת, שמתחלפת בהוגנות (לא אותו אדם כל פעם).
- **סבב = חודשיים** — כל סבב שמור עם תאריך התחלה + **תאריך החלפה** (התחלה + חודשיים). המסך מציין כמה נשאר עד ההחלפה ומתריע 🔁 כשהגיע הזמן.
- **היסטוריה** — כל סבב עם טווח התאריכים והקבוצות; אפשר למחוק.

## קבצים (apps-script/)
| קובץ | תפקיד |
|---|---|
| `config.gs` | מזהי לשוניות + `ROTATION_MONTHS` (=2) + מזהה ה-App DB (Script Properties) |
| `setup.gs` | הקמה חד-פעמית של הגיליון (לשוניות תלמידות/קבוצה קבועה/סבבים) |
| `lib.gs` | גישה לנתונים + אלגוריתם השיבוץ (`generatePairs_`, `pairCounts_`, `trioCounts_`) |
| `Code.gs` | שרת ה-Web App + API (apiData/apiSaveStudents/apiSaveFixed/apiGenerate/apiSaveRound/apiDeleteRound) |
| `index.html` | ממשק RTL: משתתפים + קבוצה קבועה → שיבוץ → היסטוריה |

## התקנה (כמו שאר האפליקציות — מסירה תחת חשבון המכון)
1. צור פרויקט Apps Script חדש; הדבק את כל קבצי ה-.gs + `index.html`.
2. הרץ `setup()` פעם אחת (יוצר את הגיליון, שומר id ב-Script Properties).
3. Deploy → New deployment → Web app (Execute as: Me · Access: לפי הצורך).
4. פתח את כתובת ה-/exec, הזן שמות (+ קבוצה קבועה), ושבץ.

**App DB:** גיליון "שיבוץ ללימוד בזוגות — App DB" עם לשוניות `תלמידות` (שם לשורה), `קבוצה קבועה` (שם לשורה), ו-`סבבים` (מזהה / תאריך התחלה / תאריך החלפה / זוגות-JSON / קבוצה קבועה-JSON).
