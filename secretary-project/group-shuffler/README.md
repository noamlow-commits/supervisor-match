# שיבוץ תלמידות לקבוצות

תת-פרויקט של `secretary-project/`. אפליקציית web פשוטה (Apps Script + Google Sheet) שמקבלת רשימת שמות תלמידות, ומשבצת אותן אחת לכמה פגישות לקבוצות קטנות של **2 או 4**.

## מה היא עושה
- **רשימת תלמידות** — מזינים שם בכל שורה, שומרים.
- **שיבוץ** — בוחרים גודל קבוצה (זוגות / רביעיות) → "שבץ" → רואים את הקבוצות → "ערבב שוב" או "שמור סבב".
- **שיבוץ חכם** — האלגוריתם מעדיף להרכיב תלמידות שעדיין לא היו יחד (לפי היסטוריית הסבבים), כך שעם הזמן כולן נפגשות עם כולן. (גרידי שמצמצם חזרות-זוגות.)
- **היסטוריה** — כל סבב נשמר עם תאריך, גודל והקבוצות; אפשר למחוק.
- אם המספר לא מתחלק יפה — קבוצה אחת תהיה בגודל שונה (3/5 וכו').

## קבצים (apps-script/)
| קובץ | תפקיד |
|---|---|
| `config.gs` | מזהי לשוניות + מזהה ה-App DB (Script Properties) |
| `setup.gs` | הקמה חד-פעמית של הגיליון (לשוניות תלמידות/סבבים) |
| `lib.gs` | גישה לנתונים + אלגוריתם השיבוץ (`generateGroups_`, `pairCounts_`) |
| `Code.gs` | שרת ה-Web App + API (apiData/apiSaveStudents/apiGenerate/apiSaveRound/apiDeleteRound) |
| `index.html` | ממשק RTL: רשימה → שיבוץ → היסטוריה |

## התקנה (כמו שאר האפליקציות — מסירה תחת חשבון המכון)
1. צור פרויקט Apps Script חדש; הדבק את כל קבצי ה-.gs + `index.html`.
2. הרץ `setup()` פעם אחת (יוצר את הגיליון, שומר id ב-Script Properties).
3. Deploy → New deployment → Web app (Execute as: Me · Access: לפי הצורך).
4. פתח את כתובת ה-/exec, הזן שמות, ושבץ.

**App DB:** גיליון "שיבוץ תלמידות — App DB" עם לשוניות `תלמידות` (שם לשורה) ו-`סבבים` (מזהה/תאריך/גודל/קבוצות-JSON).
