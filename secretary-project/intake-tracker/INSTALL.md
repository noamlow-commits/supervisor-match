# התקנה (אופציה ב' — מסירה) · תחת חשבון מכון רוטנברג

כל הצעדים מתבצעים **בחשבון ה-Gmail של מכון רוטנברג** (החשבון שמחזיק את הקובץ החי). אין צורך בשרת, בכרטיס אשראי או בשיתוף סיסמאות. ~15 דקות.

> בטיחות: כל הקוד **קורא** מהקובץ החי ולעולם לא **כותב** אליו. כל הכתיבה היא ל-App DB החדש בלבד.

---

## שלב 1 — יצירת פרויקט Apps Script (3 דק')
1. היכנסו ל-https://script.google.com (מחוברים כחשבון רוטנברג).
2. **New project** → שנו את השם ל-`מעקב פניות`.
3. מחקו את `Code.gs` הריק שנוצר. צרו 6 קבצים (➕ ליד Files → Script / HTML) והדביקו לתוכם את התוכן מ-`apps-script/`:
   - `config.gs`, `lib.gs`, `setup.gs`, `importer.gs`, `Code.gs`  → קבצי **Script**
   - `index.html` → קובץ **HTML** (➕ → HTML, בשם `index`)

## שלב 2 — חיבור והקמת ה-App DB (3 דק')
4. ב-`config.gs` ודאו ש-`LIVE_FILE_ID` הוא ה-id של הקובץ החי (כבר ממולא). השאירו `APP_DB_ID = ''` בריצה הראשונה.
5. בחרו פונקציה `setup` בסרגל העליון → **Run**. אשרו הרשאות (Review permissions → בחרו חשבון רוטנברג → Allow).
6. פתחו **Executions / Logs** (View → Logs). תראו שורה:
   `var APP_DB_ID = '....';` — העתיקו את ה-id והדביקו אותו ב-`config.gs` במקום הריק. שמרו (Ctrl+S).

## שלב 3 — ייבוא ראשון מהקובץ החי (2 דק')
7. בחרו פונקציה `runImport` → **Run**.
8. ב-Logs תראו: `ייבוא הושלם — נוספו X, עודכנו Y, דולגו Z`.
9. פתחו את ה-App DB (הקישור מודפס ב-Logs של `setup`) → לשונית **פניות** מלאה. בדקו שהתוכניות (= הלשוניות מהקובץ) והשדות נראים נכון.

## שלב 4 — פרסום האפליקציה (3 דק')
10. **Deploy → New deployment → ⚙ → Web app**.
    - Description: `מעקב פניות`
    - Execute as: **Me** (חשבון רוטנברג)
    - Who has access: **Anyone within [הארגון]** אם יש Workspace, אחרת **Anyone with the link** (ושמרו את הקישור פרטי לצוות).
11. **Deploy** → אשרו → העתיקו את ה-Web app URL. זה הקישור לטל/מיכל/צוות.

## שלב 5 — שימוש שוטף
- **לוח** — קנבן לפי שלב במשפך, סינון לפי תוכנית, דגל צהוב = צריך תזכורת. לחיצה על כרטיס → עריכה.
- **דשבורד** — משפך לכל תוכנית, רשימת תזכורות, תשלומים פתוחים.
- **⟳ ייבוא מהקובץ החי** — מסנכרן שינויים שטל עשתה בקובץ הישן (כל עוד אנחנו ב-Phase 1).
- **✉ שלח חומר** — שולח את מייל "כח הצמצום" ומסמן נשלח (מחשבון רוטנברג).

### אופציונלי — ייבוא אוטומטי יומי
ב-Apps Script: **Triggers (⏰) → Add Trigger** → function `runImport`, Time-driven, Day timer, שעה לבחירתכם.

---

## הערות מעבר (Phase 1 → cutover)
- כרגע (**Phase 1**) הקובץ הישן הוא מקור-האמת; האפליקציה מסנכרנת ממנו ומוסיפה תצוגה. עריכות באפליקציה נשמרות ב-App DB; הרצת ייבוא חוזרת לא דורסת שדות-הרחבה (תזכורות/הערות).
- כשנעבור ל**Phase 2/3** (עבודה דרך האפליקציה בלבד) — נכבה את הייבוא ונסמן את App DB כ-master. אין מיגרציה כפולה: אותה טבלת `פניות` כבר מחזיקה הכול.

## פתרון תקלות
- **"APP_DB_ID ריק"** — לא הדבקתם את ה-id מ-Logs (שלב 6).
- **לשונית בקובץ לא יובאה** — ודאו שיש בה שורת-כותרת עם לפחות 2 מהשמות המוכרים (שם/טלפון/מייל/סטטוס). לשוניות אנשי-קשר/מרצים מדולגות בכוונה (`SKIP_TABS` ב-config).
- **עמודה לא מופתה** — הוסיפו את שם-הכותרת מהקובץ למילון `SOURCE_SYNONYMS` ב-`config.gs` תחת ה-field המתאים, והריצו `runImport` שוב.
