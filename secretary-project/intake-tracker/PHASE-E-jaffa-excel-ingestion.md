# Phase E — קליטת ייצוא ה-Excel מג'אפא (CRM)

סטטוס: **בבירור — ממתינים לתשובות טל + דוגמת קובץ ייצוא.** (נדחה לסוף בכוונה; החלטת נועם 2026-06-21.)

## ההקשר
- **ג'אפא** = מערכת ה-CRM הרשמית של המכון (הרישום הרשמי + תשלומים דרך Cardcom מעוגנים שם). אין לנו גישת API אליה.
- **האפליקציה** (מעקב פניות) = מקור-האמת למשפך/מעקב מאז ה-cutover (2026-06-22).
- ג'אפא יודעת **לייצא Excel**. הרעיון: לשאוב את הייצוא הזה לאפליקציה (insert-only) כדי לחסוך הקלדה כפולה.

## יתרונות
- פונה חדש שכבר בג'אפא → פרטים (שם/טלפון/מייל/קוד לקוח) נכנסים לבד, בלי הקלדה.
- הצלבת תשלומים: לוודא שמה שרשום בקארדקום/ג'אפא תואם לסימון באפליקציה.
- פחות הקלדה כפולה.

## חסרונות / מגבלות
- ג'אפא **לא** מכילה את שלבי המשפך (ריאיון, מסמכים, התלבטויות, סטטוס) — אלה חיים רק באפליקציה. הייצוא יעזור בעיקר לשכבת **זהות + תשלומים**, לא למעקב עצמו.
- האפליקציה היא master → שאיבה ממערכת אחרת חייבת התאמה זהירה (לפי `קוד לקוח`/טלפון) כדי לא ליצור כפילויות או לדרוס עריכות. צריך להישאר **insert-only / merge-זהיר**, לא overwrite.
- כיוון אחד בלבד (Excel → אפליקציה); אין כתיבה חזרה לג'אפא (אין API).
- תלוי בפורמט הייצוא של ג'אפא — אם משתנה, צריך תחזוקה.

## אוטומציה — בלי העלאה ידנית כל פעם
מהפשוט לטוב ביותר:

1. **תיקיית Drive משותפת + טריגר זמני** (הכי ריאלי): טל שומרת את קובץ הייצוא בתיקיית Drive ייעודית; טריגר Apps Script (פעם ביום) מוצא את הקובץ החדש, ממיר ל-Google Sheet (Drive API) וקולט insert-only. הפעולה היחידה = "שמירה בתיקייה".
   - אם ג'אפא מייצאת רק ל-Downloads מקומי → **Google Drive for Desktop** מסנכרן תיקייה מקומית ל-Drive אוטומטית.
2. **דרך מייל** (הכי hands-off, אם ג'אפא תומכת): ג'אפא שולחת ייצוא מתוזמן למייל המכון → Apps Script קורא Gmail, שולף את הקובץ המצורף, קולט. אפס מגע אחרי הקמה.
3. **כפתור "העלה אקסל" באפליקציה** (ידני-קליל): הכי פשוט לבנות, אבל דורש העלאה כל פעם.

**התלות המכרעת:** איך ג'אפא יכולה להוציא את הקובץ — כפתור ידני בלבד, או ייצוא-לתיקייה / מייל מתוזמן. זה קובע אם דרך 1/2 (אוטומטי) אפשרית.

## שאלות שנשלחו לטל (וואטסאפ דרך נועם, 2026-06-22)
1. כמה פונים חדשים בשבוע, וכמה מהפרטים כבר בג'אפא?
2. מה הכי מציק להקליד מחדש?
3. האם ג'אפא יכולה לשלוח ייצוא **אוטומטית** (מייל/לוח זמנים), או רק כפתור ידני?
4. דוגמת קובץ ייצוא אחד מג'אפא (חובה — בלעדיה אי-אפשר לבנות/להעריך).

## המלצה
- אם ההקלדה הכפולה היא כאב אמיתי **ו**ג'אפא יכולה לייצא לתיקייה/מייל → לבנות דרך 1 או 2.
- אם נכנסים מעט פונים ידנית → להישאר בהזנה ידנית, בלי שכבת-סיבוך.
- **צעד הבא:** דוגמת ייצוא מג'אפא → לפיה נחליט סכמה + שיטת מיפוי (`קוד לקוח`/טלפון) + merge בטוח.
